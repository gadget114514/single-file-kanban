1. Quick Start (No kanban.json required)

Even without a kanban.json file, you can get started in just 3 steps.

1
Open kanban.html in your browser Simply double-click kanban.html in Chrome. No installation needed.
2
Create kanban.json from the template Copy kanban.template.json and save it as kanban.json. Alternatively, open Notepad and copy-paste the JSON from 7. Ready-to-Use Templates, then save it.
3
Drag & drop kanban.json onto kanban.html Drag the JSON file onto the "Open file" button at the top of the screen, or click the button to browse and select it.
Tip: Want to edit from the start? After opening kanban.json with "Open file", click the "Edit" button in the toolbar to enable adding, deleting, and moving cards directly in the browser.

3. View Mode

When you load a file, a Kanban board with columns and cards will be displayed.

ActionResult
Click a cardToggles done/undone status (green background = done)
Card left border colorPriority: ■ High / ■ Medium / ■ Low / ■ None
Overdue cardsDisplayed with red background + ⚠ icon
Done cardsDisplayed with strikethrough + light green background

Information displayed on cards

  • Title - Card name
  • Description - Displayed up to 2 lines (full details in edit modal)
  • Assignee - Initial icon + name
  • Due date - YYYY-MM-DD format. Overdue dates shown in red
  • Tags - Comma-separated labels

4. Edit Mode

Click the "Edit" button (turns green) to enable edit mode.

About first-time editing When you enable editing, a save dialog will appear to obtain write permission. Select the currently open kanban.json and click "Save". After this, changes will be auto-saved.

Card operations

Delete cardClick the ✕ button on the top right, or "Delete" in the modal
Move card (between columns)Use ◀ ▶ buttons at the bottom of the card, or drag & drop

5. Drag & Drop Operations

In edit mode, you can perform intuitive operations by dragging and dropping with the mouse.

  • Card D&D - Drag a card and drop it in another column or at a different position within a column
  • Column D&D - Drag a column header to reorder columns
Tip During D&D, a blue dashed placeholder appears at the drop destination. This indicates where the card will be inserted.

6. JSON Format

The structure of kanban.json is as follows.

{
  "board": "Project Name",
  "columns": [
    {
      "id": "column-unique-id",
      "name": "Column Name",
      "color": "#6366f1",
      "cards": [
        {
          "id": "card-unique-id",
          "title": "Card Title",
          "description": "Description text",
          "assignee": "Assignee Name",
          "priority": "high",
          "dueDate": "2026-07-01",
          "tags": ["tag1", "tag2"],
          "done": false
        }
      ]
    }
  ]
}

Field descriptions

FieldRequiredTypeDescription
boardOptionalstringBoard name (not displayed, for identification)
columnsYesarrayArray of columns
columns[].idAutostringColumn ID (auto-generated if omitted)
columns[].nameYesstringColumn name
columns[].colorOptionalstringColumn dot color (#hex, default: #6366f1)
columns[].cardsOptionalarrayArray of cards
cards[].idAutostringCard ID (auto-generated if omitted)
cards[].titleYesstringCard title
cards[].descriptionOptionalstringDescription (URLs are auto-linkified)
cards[].assigneeOptionalstringAssignee name
cards[].priorityOptionalstringhigh / medium / low / none (default: none)
cards[].dueDateOptionalstring/nullDue date (YYYY-MM-DD or null)
cards[].tagsOptionalarrayArray of tag strings
cards[].doneOptionalbooleanCompletion flag (default: false)
Tip: Minimum structure Only the columns array, each column's name, and each card's title are required. id fields are auto-generated if omitted.

9. Tips & Notes

  • Auto-save - Changes in edit mode are auto-saved after 400ms. Save status is displayed in the status bar.
  • External change detection - If kanban.json is modified by another tool, a confirmation dialog will appear when saving.
  • Language switching - You can switch the UI language with the "EN" / "日本語" button in the top right. The setting is saved in the browser.
  • URL auto-linking - URLs in card descriptions (starting with https://...) are automatically converted to clickable links.
  • Data backup - kanban.json is a plain JSON file, so you can manage it with Git or back it up by copying.
  • Offline use - No external communication is performed. Works without an internet connection.
  • Reload caution - If you close the page with unsaved changes in edit mode, a browser warning dialog will appear.
Notes
  • When opened via the file:// protocol, a file selection dialog may appear each time you edit (browser specification).
  • In environments that don't support the File System Access API (Firefox, Safari, etc.), you cannot save after editing (view only).
