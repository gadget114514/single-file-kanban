1. クイックスタート（kanban.json がなくてもOK）

kanban.json を用意していなくても、以下の3ステップですぐ始められます。

1
kanban.html をブラウザで開く Chrome で kanban.html をダブルクリックするだけで起動します。インストールは不要です。
2
テンプレートから kanban.json を作る kanban.template.json をコピーして kanban.json という名前で保存します。 または、メモ帳を開き、7. すぐ使えるテンプレート の JSON をコピー＆ペーストして保存してもOKです。
3
kanban.html に kanban.json をドラッグ＆ドロップ 画面上部の「ファイルを開く」ボタンに JSON ファイルを D&D するか、クリックしてファイル選択ダイアログから選びます。
Tip: 最初から編集したい場合 「ファイルを開く」で kanban.json を選んだ後、ツールバーの「編集」ボタンを押すと、カードの追加・削除・移動がすべてブラウザ上でできるようになります。

2. ファイルの開き方

方法説明編集可能?

ブラウザについて File System Access API に対応した Chrome / Edge での利用を推奨します。Firefox や Safari でも閲覧は可能ですが、編集時の自動保存が利用できません。

3. 閲覧モード

ファイルを読み込むと、列（カラム）とカードが並んだかんばんボードが表示されます。

操作結果
カードをクリック完了/未完了をトグル（緑背景 = 完了）
カードの左ボーダー色優先度: ■ 高 / ■ 中 / ■ 低 / ■ 無
期限超過カード赤い背景 + ⚠ マークで表示
完了カード取り消し線 + 薄緑背景で表示

カードに表示される情報

  • タイトル - カードの題名
  • 説明 - 2行まで表示（詳細は編集モーダルで確認）
  • 担当者 - 頭文字のアイコン + 名前
  • 期日 - YYYY-MM-DD 形式。期限超過は赤表示
  • タグ - カンマ区切りのラベル

4. 編集モード

「編集」ボタン（緑に変わります）を押すと編集モードが有効になります。

初回の編集について 編集をONにすると、書き込み権限の取得のため保存ダイアログが表示されます。現在開いている kanban.json をそのまま選んで「保存」を押してください。これ以降、変更は自動保存されます。

カードの操作

操作方法
カード追加各列下部の「＋カードを追加」をクリック
カード編集カード右上の ✎ ボタンをクリック → モーダルで編集
カード削除カード右上の ✕ ボタン、またはモーダルの「削除」
カード移動（列間）カード下部の ◀ ▶ ボタン、または D&D

列の操作

操作方法
列追加最右の「＋列を追加」をクリック
列名・色変更列ヘッダーの ✎ ボタンをクリック
列の並び替え列ヘッダーの ◀ ▶ ボタン、または D&D
列削除列編集モーダルの「削除」（中のカードも全て削除）

カード編集モーダルの項目

項目説明
タイトルカードの題名（必須）
説明詳細テキスト（URL は自動リンク化）
担当者担当者名
優先度高 / 中 / 低 / 無
期日YYYY-MM-DD 形式の日付
タグカンマ区切り（例: frontend, bug）

5. ドラッグ＆ドロプ操作

編集モードでは、マウスでドラッグ＆ドロップによる直感的な操作ができます。

  • カードの D&D - カードをドラッグして別の列や列内の別の位置にドロップ
  • 列の D&D - 列ヘッダーをドラッグして列の順番を変更
Tip D&D 中、ドロップ先に青い破線のプレースホルダーが表示されます。これがカードの挿入位置です。

6. JSON フォーマット

kanban.json の構造は以下の通りです。

{
  "board": "プロジェクト名",
  "columns": [
    {
      "id": "列のユニークID",
      "name": "列名",
      "color": "#6366f1",
      "cards": [
        {
          "id": "カードのユニークID",
          "title": "カードのタイトル",
          "description": "説明テキスト",
          "assignee": "担当者名",
          "priority": "high",
          "dueDate": "2026-07-01",
          "tags": ["タグ1", "タグ2"],
          "done": false
        }
      ]
    }
  ]
}

各フィールドの説明

フィールド必須型説明
board任意stringボード名（表示には未使用・識別用）
columnsYesarray列の配列
columns[].id自動string列ID（省略時は自動生成）
columns[].nameYesstring列の名前
columns[].color任意string列ドットの色（#hex、既定: #6366f1）
columns[].cards任意arrayカードの配列
cards[].id自動stringカードID（省略時は自動生成）
cards[].titleYesstringカードのタイトル
cards[].description任意string説明（URLは自動リンク化）
cards[].assignee任意string担当者名
cards[].priority任意stringhigh / medium / low / none（既定: none）
cards[].dueDate任意string/null期日（YYYY-MM-DD または null）
cards[].tags任意arrayタグ文字列の配列
cards[].done任意boolean完了フラグ（既定: false）
Tip: 最小構成 必須なのは columns 配列と各列の name、各カードの title だけです。id は省略しても自動生成されます。

9. ヒント・注意点

  • 自動保存 - 編集モードで変更すると400ms後に自動保存されます。保存状態はステータスバーに表示されます。
  • 外部変更の検知 - 別のツールで kanban.json が変更された場合、保存時に上書き確認ダイアログが表示されます。
  • 言語切替 - 右上の「EN」/「日本語」ボタンで UI 言語を切り替えられます。設定はブラウザに保存されます。
  • URLの自動リンク化 - カードの説明に https://... を書くと、自動的にクリック可能なリンクになります。
  • データのバックアップ - kanban.json はプレーンな JSON ファイルなので、Git で管理したりコピーでバックアップできます。
  • オフライン利用 - 外部通信は一切行いません。インターネット接続不要で動作します。
  • リロード時の注意 - 編集モードで未保存の変更がある状態でページを閉じると、ブラウザの警告ダイアログが表示されます。
注意点
  • file:// プロトコルで開いた場合、編集時に毎回ファイル選択ダイアログが表示されることがあります（ブラウザの仕様）。
  • Firebase / Safari など File System Access API 非対応環境では、編集後の保存ができません（閲覧のみ）。
